# Skyscanner Arabic (AR) – LQA Calibration & Style Guide

## 1. Style Guide Summary

### 1.1. Voice & Tone
- Use **Modern Standard Arabic (MSA)** — warm, accessible, not bureaucratic
- Address users with **second-person masculine singular (أنت)** unless context requires otherwise
- Write as if speaking to a friend — simple, direct, friendly
- **Creativity is key:** Translate meaning, not words; make it sound originally Arabic
- Skyscanner = **سكاي سكانر**

### 1.2. Punctuation
| Mark | Arabic Form | Usage |
|------|-------------|-------|
| Comma | ، | Between clauses (points opposite to English) |
| Question mark | ؟ | End of questions |
| Full stop | . | End of declarative sentences |
| Quotation marks | " " | Punctuation goes AFTER closing mark |

- **No space** before punctuation marks
- **Don't overuse** exclamation marks — use period/comma/colon instead where appropriate

### 1.3. Diacritics (تشكيل)
- **Avoid** unless needed for clarity
- Use for passive voice verbs on first and second-to-last letters when necessary (يُحتمَل)
- Tanween goes on letter **before** alif (مغلفًا not مغلفاً)
- Shadda is optional unless needed for disambiguation

### 1.4. Numbers & Dates
- Use **Arabic numerals** (1, 2, 3) — NOT Hindi numerals (١، ٢، ٣)
- Date format: **Day/Month/Year** (23/4/1999)
- Time: **12-hour format** preferred
- Month names: **Arabized Western** (يناير، فبراير، مارس) — NOT Levantine (كانون، شباط، آذار) or North African (غشت، يوليه)

### 1.5. Plurals & Number Agreement
Arabic has complex plural rules based on number ranges:

| Count | Form | Example |
|-------|------|---------|
| 1 | Singular | رحلة واحدة |
| 2 | Dual | رحلتان/رحلتين |
| 3-10 | Plural (جمع قلة) | رحلات |
| 11+ | Singular (تمييز) | رحلة |

**For unknown placeholders**, use workarounds:
- Structure: `عدد النتائج: {{0}}` or `{{0}} من النتائج`
- Predictable small ranges: Use جمع قلة (e.g., مقاعد for car seats)

### 1.6. Gender Handling
- Default: **Masculine** for general audiences
- For gender-neutral writing:
  - Use nominal sentences (nouns instead of gendered verbs)
  - Omit pronouns where meaning is preserved
  - Use passive voice cautiously
  - CTAs: Use kasra form (اشترِ) for gender-neutral imperatives

### 1.7. Toponyms & Names
- **Known places:** Use common Arabic version (البندقية for Venice)
- **Unknown places:** Keep in English
- **City names:** Only translate if common Arabic name exists
- **Brand names:** Keep in English (except Skyscanner → سكاي سكانر)
- **URLs:** Always keep in English

## 2. LQA CALIBRATION: FALSE POSITIVE PREVENTION

**CRITICAL:** This section defines what the evaluator must **ACCEPT**. Flagging these creates unnecessary validation burden.

---

### 2.1. Placeholder & Number Handling Tolerance

Arabic's complex number agreement makes literal placeholder translations often impossible. **Accept creative restructuring.**

#### 2.1.1. Acceptable Workarounds for Number Placeholders

| Source | Acceptable Arabic Approaches |
|--------|------------------------------|
| `{{0}} results` | `عدد النتائج: {{0}}` OR `{{0}} من النتائج` |
| `{{0}} flights` | `رحلات طيران: {{0}}` OR `{{0}} رحلة طيران` |
| `{{0}} nights` | `{{0}} ليالٍ` OR `عدد الليالي: {{0}}` |

**Do NOT flag** if the translator:
- Restructures to avoid number-noun agreement issues
- Uses `عدد الـ` (number of) + noun construction
- Uses `من الـ` (of the) construction
- Places the placeholder in a position that works for all number forms

#### 2.1.2. Do NOT Flag as "Grammar Error"
- Plural form choices for unknown placeholder values
- Singular form used where translator anticipated 11+ values
- جمع قلة used where translator anticipated 3-10 values

**Only flag** if the structure would be **incorrect for ALL possible values**.

---

### 2.2. Gender Agreement Tolerance

Gender agreement with placeholders (names, user references) is inherently challenging.

#### 2.2.1. Acceptable Gender Strategies

| Strategy | Example |
|----------|---------|
| Masculine default | Standard for general audiences |
| Passive voice | `تم حجز رحلة بواسطة {{user}}` instead of `حجز {{user}} رحلة` |
| Nominal sentences | Using nouns instead of gendered verbs |
| Omitting pronouns | `بعض الملاحظات عن الرحلة` instead of `عن رحلته/رحلتها` |

**Do NOT flag:**
- Masculine form for unknown-gender placeholders
- Passive voice used to avoid gender agreement issues
- Restructuring that avoids gendered verbs/pronouns

**Only flag** if gender is **explicitly specified** in context and translation contradicts it.

---

### 2.3. Meaning-Based Translation Tolerance

The Style Guide explicitly encourages creative, non-literal translation.

#### 2.3.1. Do NOT Flag as "Mistranslation"
- Restructured sentences that preserve meaning
- Idiomatic adaptations appropriate for Arabic
- Omission of elements that sound unnatural if translated literally
- Addition of elements required for natural Arabic expression

#### 2.3.2. Do NOT Flag as "Omission/Addition"
- Articles, pronouns, or connectors adjusted for Arabic flow
- Implicit subject/object when clear from context
- Cultural adaptations that convey the same message

**Only flag** if core meaning is changed or lost.

---

### 2.4. Punctuation & Formatting Tolerance

#### 2.4.1. Do NOT Flag
- Minor variations in comma/period placement
- Arabic (،) vs English (,) comma — both may appear in mixed text
- Presence/absence of diacritics (unless creating ambiguity)
- Different quotation mark styles

#### 2.4.2. Only Flag If
- Question becomes statement (؟ removed from question)
- Punctuation creates meaning confusion

---

### 2.5. Terminology & Synonym Tolerance

| English | Acceptable Arabic Options |
|---------|---------------------------|
| Flight | رحلة طيران، رحلة جوية |
| Book | احجز، قم بالحجز |
| Search | ابحث، بحث |
| Deal | عرض، صفقة |
| Trip | رحلة، سفرة |
| Hotel | فندق |
| Save | احفظ، وفّر (context-dependent) |

**Do NOT flag** reasonable synonym choices unless they change meaning.

---

### 2.6. Transliteration & Name Handling Tolerance

#### 2.6.1. Do NOT Flag
- Known toponyms in their common Arabic form
- Unknown places kept in English
- Brand/airline names in English
- Transliterated names that follow Arabic phonetic conventions

#### 2.6.2. Only Flag If
- Well-known Arabic name exists but English was used
- Transliteration is misspelled or unrecognizable

---

### 2.7. Severity Calibration

| Severity | Apply When |
|----------|------------|
| **Critical** | Placeholder corruption, complete meaning inversion, broken tags |
| **Major** | Core meaning changed, factual errors, incorrect directional logic |
| **Minor** | Style preferences, minor inconsistencies within acceptable range |
| **Do Not Flag** | Issues covered by tolerance rules 2.1–2.6 |

---

### 2.8. Pre-Flag Checklist

Before flagging, verify:

1. ☐ Is this a **creative restructuring** to handle Arabic grammar constraints?
2. ☐ Would the translation work for **all possible placeholder values**?
3. ☐ Is the **core meaning preserved** despite structural changes?
4. ☐ Does the Style Guide **explicitly prohibit** this choice?
5. ☐ Would a native Arabic speaker find this **incorrect** (not just different)?

**If meaning is preserved AND the structure handles Arabic grammar constraints → DO NOT FLAG**
